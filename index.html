üî® Prompt Crafting with CRAFT

Learn how to write better prompts that yield more relevant, high-quality AI responses using the structured CRAFT framework.

üí° Why It Matters

  • Most disappointing AI results are due to vague or context-free prompts.
  • Structured prompts produce better outputs with less rework.
  • Good prompting principles work across all AI tools, but we'll focus on Microsoft Copilot for university work.

"Think of prompting like a briefing: the better you brief the AI, the better its draft. But also ‚Äî the first prompt is rarely the final answer. Good prompts invite a conversation, not just a command."

üß† C.R.A.F.T Framework

Letter Meaning Example
CContext"This data is from Curtin engineering units over 3 semesters."
RRole"You are a senior Teaching Support Officer."
AAction"Analyse enrollment trends and resource utilization."
FFormat"Write in bullet points."
TTone/Target"Use clear language for a faculty board presentation."

"You don‚Äôt have to use every element every time ‚Äî but this gives you the scaffolding."

üó£Ô∏è Prompting as a Conversation with Copilot

"CRAFT is a great starting point ‚Äî but don't stop there. One-shot prompting can feel transactional. The real power is in using Copilot as a collaborator."

"Think of CRAFT as your first message to Copilot. The next steps are to discuss, clarify, iterate."

Example Chain in Copilot:

  1. Craft a CRAFT-style analysis prompt in Excel.
  2. Follow up: "Add any units at risk of cancellation."
  3. Follow up: "Now create a chart to visualize this data."

üìö Prompting Guides & Examples

Explore a collection of prompt patterns, techniques, and examples to enhance your AI interactions for various teaching support tasks.

Value-Adding Prompts

Summary Use Case (Excel with Copilot): "Analyze the unit performance data in this spreadsheet. Create a summary focusing on enrollment trends, student outcomes, and resource utilization. Add a chart showing key metrics."

Communication Use Case (Word with Copilot): "Using this unit analysis data, draft a faculty newsletter update. Include sections on teaching innovations, student satisfaction improvements, and resource efficiency. Use professional language with a positive tone."

Executive Summary (Word with Copilot): "Create an executive summary document for the board based on this semester's data. Structure it with: 1) Key achievements, 2) Compliance status, 3) Efficiency metrics, 2) Student success indicators. Use formal academic language."

Strategy Use Case (Excel with Copilot): "Analyze this unit performance data and identify the top 3 performing units. Create a table showing what makes them successful and how these practices could benefit other faculties."

Prompt Chaining Techniques

Document Analysis Workflow (Word with Copilot): First prompt: "Analyze this course proposal and identify key compliance requirements based on TEQSA standards." Follow-up: "Now highlight 3 potential compliance risks and suggest improvements for each." Final: "Create an executive summary of the analysis with clear recommendations."

Format Shifting (Word/Outlook with Copilot): First prompt: "Summarize this impact analysis report into 5 key points for the board." Follow-up: "Now draft an email to faculty staff with the same information but in a more conversational tone." Final: "Create a brief 2-paragraph update for the student portal focusing on positive outcomes."

Data Analysis Chain (Excel with Copilot): First prompt: "Analyze enrollment trends in this data." Follow-up: "Identify units at risk of low numbers." Final: "Create a visualization showing the trends with recommendations."

Advanced AI Prompting Tips

AI Debate Technique: Have two AIs critique each other's solutions.

Different AIs for Different Roles: Assign specialized roles (Compliance Officer AI, Student Success Analyst AI).

Chain-of-Thought Programming: Guide AI step-by-step through complex problems.

Reverse Engineering: Ask AI to explain existing practices/technologies.

Progressive Disclosure: Start minimal, gradually add context.

Persona-Based Prompting: "As a TSO...", "As a compliance officer...".

Prompt Playbook Highlights

One-Shot Starters: "Analyze this enrollment data.", "Draft a business case for this unit."

Multi-Turn Chains: "1. Analyze data. 2. Identify risks. 3. Create recommendations."

CRAFT-Based: "This spreadsheet contains unit metrics... Analyze as a TSO would... Identify trends... Create a summary table..."

Reflection Prompts: "What TSO task could Copilot help automate?"

üîí Secure & Sustainable AI Use

Grounding our AI exploration in responsible use is crucial. This section covers how to use AI confidently without risking sensitive data, and how to think long-term about secure, governed AI integration.

üìÑ Framing Statement

‚ÄúEven if the AI gets the right answer, if you use the wrong data or tool, it can create serious risk. Let‚Äôs talk about what‚Äôs safe, what‚Äôs smart, and what‚Äôs sustainable in your setting.‚Äù

üîç Key Principles

  1. Use mock or public data in all experiments (especially with public tools).
  2. Assume anything entered into a public AI chat is public.
  3. Use your secure platforms (like M365 Copilot) for sensitive work.
  4. Prototype in chat tools, deploy in secure environments.
  5. Governance is a team sport: involve IT/security.

ü§ñ AI Tools: What‚Äôs Under the Hood?

  • Copilot / ChatGPT: LLMs trained to predict useful text. Copilot is integrated into M365; ChatGPT is public-facing unless restricted.
  • Prompt Engineering: Structuring your input (Context, Role, Action, Format, Task - CRAFT) to guide AI output.
  • Agents: Tools that reason and act in steps (e.g., AutoGPT). Conceptually similar to complex prompt chains.
  • Key Concepts: Context Window (AI's short-term memory), Temperature (creativity/randomness), AI predicts, not "thinks".
  • Security & Risk: Copilot data stays in your M365 tenant. Avoid client-sensitive data in public ChatGPT. Always verify AI output.
  • MCPs (Most Capable Prompt): Prompts that instruct AI to simulate an expert role (e.g., "You are a senior farm consultant...").

‚öñÔ∏è Core Tensions in AI-Assisted Work

AI is powerful, but it introduces trade-offs. This section highlights key tensions to consider for balancing ambition with caution.

üß† Thinking vs. Replacing

Tension: "AI thinks for me" vs. "AI replaces my work"

What to Remember: Use AI to support reasoning, not bypass it. It's a partner, not a substitute.

üéØ Prompting vs. Collaborating

Tension: One-shot prompting vs. Over-engineering prompts

What to Remember: Most good outcomes need back-and-forth. CRAFT is a start, not the finish.
